# Event 1 Report: Learning Mindset, Prompt Engineering, and AI-Ready Fresher

### Event Objectives

The event was organized to help students better understand how to learn, work with AI, and develop themselves in a fast-changing technology environment. The content was not only about technical knowledge, but also focused on learning mindset, communication with AI, and how a fresher should prepare to adapt in the AI era.

Through the sessions, I learned more about how to build long-term learning motivation, how to write effective prompts, how to use AI as a supporting tool instead of depending on it completely, and why building projects requires clear processes, structure, and technical thinking.

### Speakers

- **Huynh Hoang Long** - Topic: Addicted to Learning Like You're Addicted to Social Media
- **Nguyen Tuan Thinh** - Topic: Automated Prompt Engineering: Enhancing LLM Output Quality
- **Duy** - Topic: AI-Ready Freshers Skills and Mindset in the AI Era

### Key Highlights

#### Addicted to Learning Like You're Addicted to Social Media

Huynh Hoang Long's sharing focused on how to make learning more attractive, similar to the way people are often drawn into social media or games. One point I found interesting was the difference between learning and social media in terms of reward mechanisms.

When studying, results often come slowly and require a long time before progress becomes visible. In contrast, social media gives almost instant rewards such as likes, short videos, new content, or continuous entertainment. Games use a similar mechanism by providing missions, rewards, levels, and achievements to keep players engaged.

From this, I learned that if I want to maintain long-term learning, I need to redesign my learning process to make it more motivating. I can divide goals into smaller tasks, reward myself after completing a learning session, create clear feedback after each task, and focus on the feeling of progress instead of only looking at the final result.

#### Automated Prompt Engineering: Enhancing LLM Output Quality

Nguyen Tuan Thinh's presentation helped me understand more about **Prompt Engineering** and how to communicate effectively with AI models. An important point is that working with AI is not simply asking a question. It requires clear instructions, context, input data, and a specific output format.

The speaker emphasized that generic prompts usually produce generic outputs. Ambiguous instructions can lead to inconsistent answers, while unnecessarily long prompts increase token usage and cost. Therefore, a good prompt needs a clear structure.

An effective prompt usually includes components such as **Role**, **Instruction**, **Context**, **Input Data**, **Output Format**, **Examples**, and **Constraints/Guidelines**. For example, when asking AI to improve a self-introduction, I should tell AI its role, the task, my background, the input content, the expected style, the word limit, and any constraints.

I also learned several prompting principles: be clear and specific, use directive language, use delimiters for long input, describe what the AI should do instead of only saying what it should not do, allow "I don't know" responses when uncertain, and break complex requests into smaller steps.

The session about tokens also helped me understand that tokens are the text processing units used by AI, and cost can depend on the number of input and output tokens. This reminds me to write prompts that are clear enough but not unnecessarily long.

#### AI-Ready Freshers Skills and Mindset in the AI Era

Duy's sharing focused on the skills and mindset that a fresher needs in the AI era. In my opinion, the most important point is that students must keep learning, adapt to change, and understand that AI is a supporting tool, not a replacement for human thinking.

A fresher should understand the employer's perspective, know how to ask questions, verify AI-generated results, and identify personal strengths. If a student only uses AI to generate code without understanding how the code works, it will be difficult to grow and solve real problems later.

From this session, I realized that I need to improve both technical skills and thinking skills. When using AI, I should not copy results mechanically. Instead, I need to read, understand, test, and adjust them to fit my project.

#### Don't Just Vibe Code, Build With Software Engineering Standards

Another important message was that building a project should not stop at making code run. In real working environments, projects need clear processes, proper structure, maintainable code, and quality control.

AI can help generate ideas, suggest structure, write sample code, or support debugging, but humans still need to coordinate the overall direction. A project builder must understand requirements, design workflows, organize folders, use clear naming, test features, and make sure the project can be developed further.

The speaker also introduced **BMAD-METHOD** as an approach for building projects with better structure and process. This helped me understand that professional project development requires organization, documentation, requirement analysis, and quality control, not just fast coding.

### What I Learned

#### About Learning Mindset

- Learning should be divided into smaller goals to make it easier to maintain.
- Self-reward and feedback after each study session can increase motivation.
- The feeling of making small progress is more important than only focusing on the final result.
- Understanding the reward mechanisms of social media and games helps me adjust my learning habits.

#### About AI Skills And Prompt Engineering

- A good prompt should be clear, contextual, and specific about the expected output.
- Role, instruction, context, input, examples, and constraints should be provided when needed.
- Vague prompts can produce inaccurate or inconsistent results.
- AI-generated results should be checked instead of trusted completely.
- Understanding tokens helps use AI more efficiently and economically.

#### About Fresher Mindset

- Freshers need the ability to self-learn and adapt continuously.
- Students should not depend completely on AI when learning or building projects.
- It is important to ask questions, verify information, and understand personal strengths.
- Technical skills should go together with problem-solving thinking.

#### About Software Engineering

- A good project is not only a project that runs, but also one with a clear structure.
- Code should be readable, maintainable, and extendable.
- AI should be used as a supporting tool, while humans still control the direction.
- Process, documentation, testing, and overall quality are important in project development.

### Applying The Lessons To Myself

After the event, I can apply these lessons to my learning and project development. When studying a new topic, I should divide goals into smaller parts, create a clear plan, and track my progress to maintain motivation. Instead of studying only when I feel inspired, I should build a steady learning habit and recognize progress through small achievements.

When using AI, I need to write clearer prompts, provide enough context, and specify the expected output. I also need to check AI-generated results, avoid copying code mechanically, and use AI to support thinking, project organization, or solution ideas. For personal projects, I should pay more attention to structure, documentation, process, and maintainability.

### Event Experience

For me, the event provided practical and useful knowledge for students, especially those preparing for internships or starting their career direction in technology. The sessions were closely related to common student problems such as lack of learning motivation, ineffective AI usage, and project development without a clear process.

I was impressed by the learning mindset session because it helped me reflect on how I study and why social media often feels more attractive than learning. The Prompt Engineering session helped me understand that if I want AI to answer well, I also need to communicate well. The AI-ready fresher and software engineering sessions reminded me that in the AI era, learners need to understand fundamentals, control quality, and build a more professional working mindset.

### Lessons Learned

- Learning motivation can be built by dividing goals into smaller tasks and creating clear feedback.
- Prompt Engineering is an important skill when working with AI.
- AI is most useful when users know how to give instructions, verify, and refine outputs.
- Freshers need continuous learning, adaptability, and independence from AI.
- Projects should be built with clear processes, structure, and software engineering thinking.

### Conclusion

Through Event 1, I learned important lessons about learning mindset, Prompt Engineering, and how to prepare as a fresher in the AI era. The event helped me understand that to grow in technology, I do not only need to know how to code, but also need to learn effectively, use AI properly, verify results, and build projects with a clear process. These are important directions for improving myself during my study, internship, and future work.
